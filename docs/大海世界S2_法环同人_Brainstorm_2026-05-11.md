## 三、赐福点与房间机制

**改动方向**：大厅 → 赐福点/篝火点

- 赐福点是据点，不是 Boss 房间
- 在赐福点能做三件事：坐火休息（回血+挂机）、强化（升级）、出征（选 Boss）
- 出征流程：选 Boss → 查有没有活跃房间（战斗中且未满）→ 有则匹配 / 无则新建
- 匹配进其他玩家正在打的 Boss，保证房间不空

**进度锁**：
- 每个区域按线性顺序解锁
- 未解锁区域的 Boss 不可选、不创房
- 解锁条件：前一区域 Boss 击败 + 练度达标
- 用 `if grace_unlocked[region_id]` 在代码里防跳级

## 四、幻影系统（解决人少时房间空的方案）

玩家少的时候，打过的玩家留下**战斗回放幻影**：

- 存 MongoDB `battle_phantoms` collection
- 数据：伤害轴 + build + 用时 + 等级
- 单人进房时回放 2-3 条幻影 → Boss 身上飘别人的伤害数字
- 玩家多时幻影自动退场，切到真人匹配
- 幻影保留最近 48 小时

**额外热闹感**：Boss 血条下显示 "本小时 X 名褪色者挑战 / 总讨伐 Y 次 / 最快击杀 Z"

## 五、挂机系统改动

**彻底与战斗系统隔离**，改为纯资源产出：

- 只产出：金币、强化石
- 不产出：经验、装备、Boss 进度
- 机制：坐火时缓慢积累
- 设计目的：消除在线/离线玩家之间的不公平感

## 六、数值防崩坏方案

### 等级压制区
```
最终伤害 = 基础攻击 × 武器倍率 × 攻击力乘区 × 增伤乘区 × 等级压制系数
等级压制系数 = min(1.0, 玩家等级 / Boss等级 × 1.5)
```

### 装备等级阶梯
按区域分阶段限制强化上限，素材在当前区域产出：
- 阶段 1：武器 +3
- 阶段 2：武器 +5
- ...逐步解锁到 +10

### 动态 Boss 血量（房间人数缩放）
```
Boss 基础血量 × (1 + 0.3 × (房间内人数 - 1))
```

### 战斗韧性/架势条
每次点击削减韧性 → 打空后处决窗口（伤害 x2）→ 处决后韧性上限降低 10%

### 掉落保底
首杀 100% 掉落，后续递减（50%→30%→15%），每天重置。集碎片可兑换。
