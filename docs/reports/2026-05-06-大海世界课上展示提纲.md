## 第 4 页：总体架构设计

**页面标题**

总体架构设计

**页面文案**

- 前端使用 `Vue 3 + Vite` 构建页面与交互逻辑，负责战斗页、资料页、任务页、消息页和后台页面展示。
- 后端使用 `Go + Hertz` 提供 HTTP 接口、实时通信入口和业务逻辑处理。
- 实时通信链路以 `WebSocket` 为主，在部分场景下结合 `SSE` 进行事件推送与状态更新。
- 数据层采用 `Redis + MongoDB` 分工协作：Redis 负责热数据与高频状态，MongoDB 负责冷数据、日志、任务和归档。
- 部署层通过容器与自动化构建流程支撑项目运行和发布。

**可直接放入 PPT 的架构流向**

- 用户操作前端页面
- 前端请求后端接口 / 建立实时连接
- 后端处理业务逻辑并更新 Redis / MongoDB
- 后端再把状态变化推送回前端
- 后台系统负责内容配置、功能管理和运维支持
