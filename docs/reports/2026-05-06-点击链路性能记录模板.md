***【结论】***

- ✅ 优化效果显著，***全局 6-27% 改善***，尤其是最大延迟砍了 27%
- 最大延迟从 333ms → 241ms，尾部降幅最大，说明连接池或锁竞争有改善
- `getEquipmentDefinition` 和 `loadoutForNickname` 合计节省 ***336MB*** 是核心优化点
- GC 次数上升（540→879）但 GC CPU 占用仅增 0.18%，说明单次 GC 更快
- 剩余瓶颈：`itemInstancesByIDForNickname`（装备实例查询）+ Redis HMGet 仍占分配 60%+
