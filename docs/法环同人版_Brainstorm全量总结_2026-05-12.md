**基础框架**: hai-world S2 架构（JS 栈，带 9 万行代码+文档）  
**设计定位**: 法环（Elden Ring）同人改造版点击游戏

## 一、核心设计原则

### 1.1 魂系精神 → 点击游戏的语言转化

| 魂系元素 | 点击游戏等价物 | 说明 |
|---------|---------------|------|
| 翻滚/格挡 | 无（玩家不受击） | ❌ 玩家不承担受击反馈，不存在HP/闪避/减伤 |
| 弹反/处决 | 削韧Break + 输出窗口 | ✅ 操作精度决定收益 |
| 喝药 | 连击Combo维持资源循环 | ✅ 节奏感替代资源管理 |
| 跑图/坐火 | 无 | ❌ 纯 Boss Rush，无跑图 |
| 等级压制 | 无等级锁 | ✅ 任何玩家可挑战任何boss |
| 死亡惩罚 | 超时 = 重置 | ✅ 魂系挫败感的替代方案 |
| FP/专注值 | 战斗内资源池（精力/专注） | ✅ 技能消耗控制，高手资源利用率更高 |

### 1.2 玩家分层模型

| 层级 | 特征 | 体验方式 |
|------|------|----------|
| **高手** | 背板、精准操作、资源管理 | 1级打后期boss，sub-N速通 |
| **中间玩家** | 会一些机制但不全 | 比推荐等级低10-15级挑战 |
| **普通玩家** | 朴实点击，不追求机制 | 升级装备碾压式通关 |

### 1.3 越级击杀的设计哲学

- **核心杠杆**：机制伤害 = 百分比真伤，与玩家面板无关
- **实现方式**：削韧Break、破防连击、场地交互 三种机制提供百分比伤害
- **验证方式**：击杀时间排行榜（不标注"越级"，数据自己说话）

## 二、战斗机制系统

### 2.1 核心机制 ⭐ 架势削韧Break

**原理**：
Boss 有隐藏韧性条（低显UI）。Boss特定招式抬手时，弱点部位出现短暂高亮窗口（约1秒）。点击高亮处积累削韧值。满 → 架势崩溃 → 可处决。

**伤害公式**：
- 处决伤害 = Boss 最大 HP × 25%~35%（真实伤害，与攻击力无关）

**分层表现**：
- 高手：刻意等 boss 出招来削韧，处决打掉1/3血
- 普通玩家：不知道机制或反应慢，纯平A磨血

### 2.2 核心机制 ⭐ 破防行为·输出窗口制

**原理**：
Boss 进入不可输出状态（硬化/虚化/高速移动），必须完成特定"破防操作"才能恢复输出。

**破防操作类型**（每个 Boss 择一）：
1. **节奏连击**：屏幕浮现符文，按正确顺序点击 → Boss倒地 + 8秒爆发
2. **定位点击**：高亮区随机移动，必须点中精确位置
3. **序列记忆**：Boss 演示动作序列，玩家按同样顺序点击

**越级杠杆**：操作与玩家等级/装备无关。30级和300级面对相同判定。

### 2.3 辅助系统 ⭐ 连击断点

**原理**：
每次成功点击积累连击计数。连击不影响基础伤害，但影响资源效率。

| 连击区间 | 技能CD缩减 | 资源获取率 |
|----------|------------|-----------|
| 0~9 | 基准值 | 100% |
| 10~19 | -15% | 130% |
| 20~49 | -30% | 170% |
| 50+ | -45% | 220% |

**断连条件**：
- 点了 Boss 的假动作（佯攻——点上去断连击）
- Boss 位移期放技能 → 技能额外 CD

**越级杠杆**：越级时基础伤害不够，靠高连击加速技能循环填补面板差距。

### 2.4 辅助系统 多目标噪点

**原理**：Boss战出现干扰单位（不攻击玩家，只影响输出效率）。

| 干扰体 | 效果 | 应对策略 |
|--------|------|----------|
| 寄生体 | 趴在 boss 身上挡住弱点 | 点掉才能打到弱点 |
| 幻影 | 分身稍延迟，真身才吃伤害 | 读微动作辨真身 |
| 屏障编织者 | 给boss叠减伤buff（每层5%） | 优先清小怪 |

### 2.5 环境系统 场地交互

**原理**：每个 Boss 房间放 2~3 个交互环境物件。

**示例**：
- 吊灯/落石：红光时点击 → boss HP × 12% 真伤 + 倒地
- 岩浆闸门/陷阱：引boss到位置后触发
- 铁笼/锁链：限制boss移动，创造稳定输出窗口

**越级杠杆**：伤害含百分比部分，与面板无关。

## 三、击杀时限系统（核心判负机制）

### 3.1 基础规则

- 每个 Boss 有击杀时限，屏幕顶端显示倒计时
- 超时 = 挑战失败，Boss血重置，无额外惩罚
- 玩家1级可挑战任何Boss

### 3.2 推荐时限跨度

| 阶段 | 时限 | 设计意图 |
|------|------|----------|
| 前期boss | 5~8 min | 给新玩家容错空间 |
| 中期boss | 3~5 min | 逼迫学习1~2个核心机制 |
| 后期boss | 2~3 min | 考验操作精度+机制熟练度 |
| 隐藏boss | 无时限 | 纯DPS检验，Boss血量极高 |

### 3.3 奖惩哲学

- 超时不掉魂不跑图，点一下重开
- 不设"越级击杀"标签或稀有成就，仅有击杀时间记录
- 排行榜上的时间 = 实力的自然分层

## 四、数值框架初步构想

### 4.1 战斗内资源池（替代传统体力）

**核心逻辑**：
- 玩家可自由挑战任何Boss，没有每日限制
- 每场战斗内置消耗资源池（FP/专注/精力）
- 池子大小 = 玩家属性（等级/装备），进场满池
- 技能、爆发、破防操作消耗资源
- 部分操作（Break/处决/完美操作）可以回复少量
- 空池后只能平A，直到特定恢复阶段

**高手体验**：资源利用率90%+，在 Break 窗口期集中爆发
**普通玩家体验**：前段消耗完资源，后段平A刮痧

### 4.2 Boss 血量生长模型（草稿）

基础公式：
```
Boss(n) 血量 = BaseHP × (Growth)^(n-1)
```

待定参数：
- BaseHP（1号Boss血量）
- Growth Rate（生长系数，推荐 1.3~1.5）
- 与玩家同阶段DPS的交叉点（普通玩家纯点击能否在时限内过）

### 4.3 机制伤害占比设计

- 普通通关：机制伤害占 0%~10%（可有可无）
- 越级通关：机制伤害占 60%~80%（必须打满）
