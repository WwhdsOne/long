# 天赋系统开发参考（2026-04-26）

本文对应本次“天赋点获取 -> 学习扣点 -> 战斗生效 -> 洗点返还 -> 前后端展示”完整实现，供后续策划/开发手工调参。

## 1. 关键调用链与执行顺序

### 1.1 后端调用链

1. Boss 击杀发放资源
   1. 入口：`Store.finalizeBossKill`（`backend/internal/vote/store.go`）
   2. 行为：给达标参与者发放金币、强化石、天赋点（`talent_points`）
2. 学习天赋
   1. API：`POST /api/talents/learn` -> `talentAPI.learn`（`backend/internal/httpapi/talent_routes.go`）
   2. 存储：`Store.LearnTalent`（`backend/internal/vote/talent.go`）
   3. 行为：前置检查 -> 点数检查 -> `talents` 写入 + `talent_points` 扣减
3. 洗点返还
   1. API：`POST /api/talents/reset` -> `talentAPI.reset`
   2. 存储：`Store.ResetTalents`
   3. 行为：遍历已学节点累加成本 -> 清空已学节点 -> 返还点数
4. 战斗生效
   1. 点击入口：`Store.ClickBossPart` -> `Store.applyBossPartDamage`
   2. 静态加成：`Store.combatStatsForNickname` 内调用 `Store.ComputeTalentModifiers`
   3. 触发型效果：`Store.applyTriggeredTalentDamage`（当前实现了 `normal_core` 追击链）
5. 实时反馈
   1. 回包：`realtime_click_ack`（`backend/internal/httpapi/realtime_socket.go`）
   2. 字段：`talentEvents`、`partStateDeltas`、`bossDamage`

### 1.2 前端调用链

1. 战斗页资源展示
   1. `loadBossResources`（`frontend/src/pages/publicPageState.js`）读取 `/api/boss/resources`
   2. `BattlePage.vue` 展示金币/强化石区间与固定天赋点
2. 天赋学习/洗点
   1. `TalentsPage.vue` 调 `GET /api/talents/defs`、`GET /api/talents/state`
   2. 点击节点学习：`POST /api/talents/learn`
   3. 洗点：`POST /api/talents/reset`
3. 实时触发反馈
   1. `publicPageState.js` 的 `onClickAck(payload)` 读取 `payload.talentEvents`
   2. `BattlePage.vue` 显示“天赋触发”反馈条

## 2. 公式与代码位置

### 2.1 天赋点获取（击杀）

- 公式：`talentPoints += boss.TalentPointsOnKill`
- 代码：`Store.finalizeBossKill`（`backend/internal/vote/store.go`）
- 说明：本版本为固定值发放，不做随机浮动。

### 2.2 学习扣点

- 公式：`talentPoints -= talent.cost`
- 代码：`Store.LearnTalent`（`backend/internal/vote/talent.go`）
- 说明：先校验前置与副系限制，再校验点数。

### 2.3 洗点返还

- 公式：`refund = Σ(已学节点cost)`，`talentPoints += refund`
- 代码：`Store.ResetTalents`（`backend/internal/vote/talent.go`）

### 2.4 战斗结算（含天赋加成）

1. 静态战斗属性聚合
   1. 代码：`Store.combatStatsForNickname`（`backend/internal/vote/store.go`）
   2. 来源：装备加成 + `ComputeTalentModifiers` 结果
2. 部位伤害
   1. 代码：`CalcBossPartDamage`（`backend/internal/vote/store.go`）
   2. 包含：全局增伤、部位增伤、每存活部位增伤、低血阈值乘区
3. 触发型天赋
   1. 代码：`Store.applyTriggeredTalentDamage`
   2. 当前实现：`normal_core` 每 `triggerCount` 次点击触发追击，附带 `normal_chase_up`、`normal_combo_ext` 的参数联动

## 3. 可调参数清单

### 3.1 天赋成本

- 位置：`backend/internal/vote/talent.go` -> `talentTierCosts`
- 默认值：
  - `tier0=100`
  - `tier1=2000`
  - `tier2=8000`
  - `tier3=30000`
  - `tier4=120000`
- 影响：学习节奏、首点解锁速度、后期爬升速度。

### 3.2 Boss 天赋点掉落

- 字段：`talentPointsOnKill`
- 涉及结构：`BossUpsert`、`BossTemplate`、`BossTemplateUpsert`、`Boss`
- 影响：击杀资源增长速度，直接改变可学习密度。

### 3.3 触发型天赋参数

- 位置：`talentDefs["normal_core"].EffectValue`
- 参数：`triggerCount`、`extraHits`、`chaseRatio`
- 联动参数：
  - `normal_chase_up` -> `chaseRatio`
  - `normal_combo_ext` -> `extraHits`

### 3.4 静态伤害系数参数

- 位置：`ComputeTalentModifiers` 与 `CalcBossPartDamage`
- 参数：
  - `attack_power_percent`
  - `all_damage_amplify`
  - `part_type_damage`
  - `per_part_damage`
  - `low_hp_bonus`
- 影响：各流派对不同部位与不同血线阶段的输出曲线。

## 4. 常见调参目标与改法示例

### 4.1 目标：首点更快

1. 下调 `tier0` 成本（如 `100 -> 60`）
2. 或提高常见 Boss 的 `talentPointsOnKill`

### 4.2 目标：后期更慢

1. 上调 `tier3`、`tier4` 成本（如 `30000 -> 45000`，`120000 -> 180000`）
2. 保持 `tier0/tier1` 不变，减少前期体验影响

### 4.3 目标：抑制某流派过强（示例：普攻追击）

1. 提高 `normal_core.triggerCount`（如 `200 -> 260`）
2. 下调 `normal_core.chaseRatio`（如 `0.5 -> 0.35`）
3. 或下调 `normal_combo_ext.extraHits`

## 4. 层锁规则

详见 `docs/designs/2026-04-27-天赋树改造案.md`。

### 4.1 规则

- 主系：必须点满第 N 层所有节点（主节点 + 小节点），才能学习第 N+1 层
- 副系：不受层锁限制

### 4.2 代码改动

| 位置 | 改动 |
|------|------|
| `Store.LearnTalent`（`talent.go`） | 新增 `isTierFull(tier)` 前置校验 |
| `canLearn()`（`TalentsPage.vue`） | 新增 `layer-locked` 状态 |
| `ComputeTalentModifiers`（`talent.go`） | 新增 `computeTierCompletionBonuses` 层满奖励 |

### 4.3 相关常量

- 每系小节点数：Tier1=2, Tier2=2, Tier3=2, Tier0=0, Tier4=0
- 层满奖励见改造案第三章

## 5. 最小联调脚本与回归步骤

### 5.1 最小联调脚本（建议）

1. 启后端：`rtk go run ./cmd/server`
2. 启前端：`rtk npm --prefix frontend run dev`
3. 后台创建/编辑 Boss 模板，填 `talentPointsOnKill`
4. 前台登录后击杀 Boss，确认：
   1. 资源区天赋点增长
   2. 天赋页可学习节点状态变化
   3. 学习后点数扣减
   4. 洗点后返还

### 5.2 回归步骤

1. Boss 配置页保存后刷新，`talentPointsOnKill` 不丢失
2. `/api/boss/resources` 在旧 Boss 数据（无该字段）下返回 0
3. 点击战斗下 `talentEvents` 和 `partStateDeltas` 字段可选返回，不影响旧前端字段解析
4. 装备、公告、消息链路保持可用

## 6. 易错点与排查路径

1. 学习返回“点数不足”
   1. 看 `/api/talents/state` 的 `talentPoints`
   2. 核对 `talentDefs` 对应节点 `cost`
2. 击杀后不发天赋点
   1. 检查当前 Boss 是否有 `talentPointsOnKill`
   2. 检查是否达到 1% 参与门槛（与金币/强化石共用）
3. 天赋效果“定义有但战斗不生效”
   1. 看 `combatStatsForNickname` 是否拿到 `ComputeTalentModifiers`
   2. 看 `CalcBossPartDamage` 是否使用对应字段
4. 触发事件前端无反馈
   1. 检查 websocket `click_ack` 是否带 `talentEvents`
   2. 检查 `publicPageState.js onClickAck` 是否解析并写入 `talentTriggerFeed`
5. 旧数据兼容
   1. Boss 无 `talent_points_on_kill` 时应走默认 0
   2. 玩家资源无 `talent_points` 时应按 0 处理
