# 天赋数值平衡修复设计

**日期：** 2026-05-03
**关联策划案：** `docs/designs/2026-04-30-天赋系统数值策划案-V2.0.md`
**状态：** 待实现

## 背景

V2.0 天赋系统实装后，两个数值问题被用户反馈：

1. **均衡攻势（普攻系）整体太弱**：追击爆发伤害不足，清 Boss 速度明显慢于暴击系
2. **终末血斩（暴击 T4）倍率太低**：消耗全部 150 层死兆的收益远低于持有死兆的持续收益，数学上严重亏损

## 方案

### 修复一：追击倍率提升

**保持不变：** 公式仍为 `actualDamage × chaseRatio × extraHits`

**改动：** 提升追击倍率相关插值函数的 Lv5 上限

| 参数 | 旧 Lv1→Lv5 | 新 Lv1→Lv5 |
|------|-----------|-----------|
| 暴风连击基础倍率 (`normalCoreChaseRatio`) | 0.20→0.30 | 0.60→0.90 |
| 追击强化倍率 (`normalChaseUpgradeRatio`) | 0.80→1.20 | 2.40→3.60 |
| 追猎填充倍率 (`normalFillerT2aChase`) | 0.10→0.30 | 0.30→0.90 |

**Lv5 满配追击总倍率：** 3.60 + 0.90 = **4.50**（旧值 1.50，提升 3x）

**Lv5 满配追击爆发伤害：**
```
追击爆发 = actualDamage × 4.50 × 68段 = actualDamage × 306
等效每点击倍率 = 306 / 40(触发间隔) = 7.65x
```

**涉及函数（`backend/internal/core/talent.go`）：**
- `normalCoreChaseRatioForLevel`：Lv5 0.30 → 0.90
- `normalChaseUpgradeRatioForLevel`：Lv5 1.20 → 3.60
- `normalFillerT2aChaseForLevel`：Lv5 0.30 → 0.90

### 修复二：终末血斩倍率提升

**改动：** `critFinalCutDamageRatioForLevel` 的 Lv5 上限从 30.0 提升到 120.0

**数学验证**（Lv5 满配，150 死兆，打弱点，低血量）：

持有 150 死兆时单次暴击倍率：约 ×327
消耗后（0 死兆）单次暴击倍率：约 ×20
每点击损失：×307

```
旧血斩：327 × 30 = 9,810x  → 回本需 9,810/307 ≈ 32 次点击
新血斩：327 × 120 = 39,240x → 回本需 39,240/307 ≈ 128 次点击
```

重建 150 死兆约需 100~150 次点击，新倍率确保点出终极技能有正收益。

**涉及函数（`backend/internal/core/talent.go`）：**
- `critFinalCutDamageRatioForLevel`：Lv5 30.0 → 120.0

## 影响范围

- `backend/internal/core/talent.go`：修改 4 个 per-level 插值函数的 Lv5 上限
- 不涉及数据结构变更、不涉及 Redis 状态迁移、不涉及前端展示逻辑变更
- 前端描述文本自动通过 `TalentEffectDescriptionForLevel` 反映新数值

## 验证计划

1. 运行 `go -C backend test ./...` 确认现有测试通过
2. 查看 `combat_cache_test.go` 中天赋编译测试是否需要更新期望值
3. 部署后观察普攻系与暴击系的 Boss 击杀时间是否趋于合理差距
