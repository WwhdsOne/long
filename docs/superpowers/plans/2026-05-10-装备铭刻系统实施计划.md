# 2026-05-10 装备铭刻系统实施计划

## 目标

在不影响现有强化、分解、穿戴链路的前提下，为装备实例新增“铭刻”能力，并把刻印石接入 Boss 掉落、玩家资源、前端右键菜单和 tooltip。

## 实施分解

### 1. 后端数据层

- 在 `ItemInstance` 新增 `affixes`
- 在 `InventoryItem` 新增 `affixes / affixCount / affixLimit`
- 在资源读取链路新增 `inscriptionStones`
- 在 `Boss` / `BossResources` / `BossUpsert` / `BossTemplate` 新增刻印石掉落字段

### 2. 后端逻辑层

- 新增 `maxAffixCount(rarity)`，独立于 `maxEnhanceLevel(rarity)`
- 新增铭刻词条池与敏感词条封顶规则
- 新增 `InscribeItem(ctx, nickname, instanceID)`：
  - 校验实例归属
  - 校验刻印石数量
  - 校验词条未满
  - 生成词条
  - 写回实例
  - 扣减 `inscription_stones`
  - 刷新 `combatStats`

### 3. Boss 奖励链路

- Boss 结算时单独结算刻印石
- 当前公共资源接口返回：
  - 掉率
  - 掉落数量区间

### 4. HTTP 接口

- 新增 `POST /api/equipment/:instanceId/inscribe`
- 错误映射：
  - `INSCRIPTION_STONE_NOT_ENOUGH`
  - `EQUIPMENT_AFFIX_LIMIT_REACHED`

### 5. 前端状态

- `publicPageState.js` 新增 `inscriptionStones`
- 新增 `inscribeItem(instanceId)` 请求方法
- 玩家资料同步时读取 `inscriptionStones`

### 6. 前端交互

- 背包右键菜单新增 `铭刻`
- 新增确认弹窗，明确：
  - 消耗 `刻印石 x1`
  - 随机新增 `1` 条词条
  - 永久绑定、无法撤销
  - 当前词条数 / 上限
- Tooltip 展示词条数量与词条列表

## 测试顺序

1. 后端核心单测
2. HTTP 路由单测
3. 前端静态断言测试
4. `go test` 相关包回归
5. `bun --cwd=frontend run test`

## 上线注意

- 旧装备实例没有 `affixes` 字段时按空数组处理
- 旧玩家资源没有 `inscription_stones` 字段时按 `0` 处理
- 首版不做迁移脚本
- 若后台后续需要编辑刻印石掉落参数，再单独补管理入口
