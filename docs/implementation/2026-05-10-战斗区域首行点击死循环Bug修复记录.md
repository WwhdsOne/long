# 战斗区域首行点击死循环 Bug 修复记录

**日期**：2026-05-10
**分支**：dev
**影响范围**：Boss 战斗区域 5x5 网格点击响应

## 现象

- 战斗区域有连击状态时，浏览器控制台持续报错：
  ```
  Maximum recursive updates exceeded in component <BattlePage>.
  This means you have a reactive effect that is mutating its own dependencies
  and thus recursively triggering itself.
  ```
- 触发路径一：`nowTick.value++`（每 250ms）→ `battleEffectEntries` computed → DOM 测量 → 响应式写入 → 死循环
- 触发路径二：`comboTimeoutPercent.value`（每 200ms）→ `globalStatusList` computed → 模板重渲染 → 同上死循环

## 根因

`battleEffectEntries` 是一个 Vue `computed` 属性，在求值过程中存在 **6 处 `getBoundingClientRect()` 调用**，它们分别通过以下函数链进入 computed 求值路径：

```
battleEffectEntries computed (依赖 nowTick, 每 250ms 重算)
  ├─ effectEntryLayout()
  │   ├─ triggerAnchor()        → getBoundingClientRect() × 2  ← 强制同步重排
  │   ├─ gridOverlayAnchor()    → getBoundingClientRect() × 2  ← 强制同步重排
  │   ├─ bossGridEffectSize()   → getBoundingClientRect() × 1  ← 强制同步重排
  │   └─ resolveFallbackAnchor()
  │       └─ measureTalentOverlaySize()  → getBoundingClientRect() + 修改响应式 ref
  └─ measureTalentOverlaySize() → getBoundingClientRect() + 修改 talentOverlayCanvasSize.value
```

这些 `getBoundingClientRect()` 调用**强制浏览器同步重排**。如果重排过程中触发 ResizeObserver 回调 → 修改 `bossCellSizePx` 和 `talentOverlayCanvasSize`（两者都是 `battleEffectEntries` 的响应式依赖）→ computed 再次重算 → 再次强制重排 → **死循环**。Vue 在 100 次递归后抛出 `Maximum recursive updates exceeded`。

### 关键违规

1. **computed 内修改响应式状态**：`measureTalentOverlaySize()` 在 computed 求值路径中写入 `talentOverlayCanvasSize.value`
2. **computed 内调用 `getBoundingClientRect()`**：`triggerAnchor`、`gridOverlayAnchor`、`bossGridEffectSize` 都在 computed 求值路径中强制同步重排
3. **ResizeObserver 与 computed 形成闭环**：ResizeObserver 回调修改 `bossCellSizePx` → computed 依赖 `bossCellSizePx` → computed 触发重排 → 可能同步触发 ResizeObserver

## 修复方案

将所有 DOM 测量从 computed 求值路径中移除，统一收敛到 ResizeObserver 和 onMounted：

1. **新增 `refreshCachedLayout()`**：统一测量 overlay rect、grid rect、所有 zone 元素 rect，并更新 `bossCellSizePx` 和 `talentOverlayCanvasSize`。仅在 `onMounted` 和 `ResizeObserver` 回调中调用。

2. **`measureTalentOverlaySize()` 改为纯 getter**：不再调用 `getBoundingClientRect()`，只返回 `talentOverlayCanvasSize.value`

3. **`triggerAnchor()` 改用缓存**：使用 `cachedOverlayRect` 和 `cachedZoneRects` 替代 `getBoundingClientRect()`

4. **`gridOverlayAnchor()` 改用缓存**：使用 `cachedOverlayRect` 和 `cachedGridRect` 替代 `getBoundingClientRect()`

5. **`bossGridEffectSize()` 改用缓存**：使用 `cachedGridRect` 替代 `getBoundingClientRect()`

6. **`onMounted` / ResizeObserver 统一调用 `refreshCachedLayout()`**

### 修改文件

- `frontend/src/pages/BattlePage.vue`：重构 DOM 测量体系
- `frontend/src/pages/BattlePage.talentFx.test.js`：更新静态分析测试以匹配新函数名

## 验证

- 前端测试：64 文件 256 测试全部通过
- 前端构建：成功
- `getBoundingClientRect()` 调用已全部移出 computed 求值路径（仅保留在事件处理器和 ResizeObserver 中）
- ResizeObserver → computed → ResizeObserver 死循环已断开
