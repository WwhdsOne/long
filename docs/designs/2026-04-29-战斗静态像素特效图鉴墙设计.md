# 战斗静态像素特效图鉴墙设计

日期：2026-04-29

## 1. 背景

当前 Boss 战斗页的天赋触发特效混用了 OSS 贴图、序列帧贴图与少量原生像素表现。现阶段的问题不是单张图是否足够精细，而是视觉表达方式不统一：部分动作类特效依赖描点贴图或 CSS 放缩伪造动态，导致像素风格不稳定、层级拥挤、并发触发时相互遮挡。

本次工作不直接改线上战斗页，也不替换真实触发逻辑，先制作一个内部 Demo 页面，用于评审“全部改为静态像素图”后的战斗特效方向。

## 2. 本次目标

本次只完成一个内部 Demo 页面，目标如下：

1. 停止把 OSS 战斗特效资源作为 Demo 视觉来源。
2. 所有 Demo 特效统一改为静态像素画风格。
3. 先用图鉴墙逐个对比当前已接线的战斗特效。
4. 额外提供多特效并发组合预览，验证最终效果不互相覆盖。

## 3. 非目标

以下内容明确不在本次范围内：

1. 不修改 BattlePage 真实触发链路。
2. 不接后端，不消费实时事件。
3. 不替换现有 OSS 资源配置。
4. 不做动画，不做序列帧，不做 Canvas 爆散。
5. 不处理尚未接入 BattlePage 的未来特效。

## 4. 展示方式

页面采用“单特效图鉴墙 + 并发组合预览”双区结构。

### 4.1 单特效图鉴墙

每种已接线特效各占一张卡片，使用统一的 Boss 格子底板进行展示，确保对比公平。每张卡片固定包含：

1. 特效名称
2. 对应触发类型
3. 静态像素特效预览
4. 设计说明
5. 建议层级

这里不模拟真实点击或倒计时，只看静态构图是否成立。

### 4.2 并发组合预览

页面底部单独提供一块组合区域，用 1 组放大的 Boss 格子预览多特效同时存在时的层级关系。组合区至少包含以下组合：

1. `collapse_trigger` + `auto_strike`
2. `doom_judgment` + `final_cut` + `crosshair-mark`
3. `storm_combo` + `silver_storm` + `bleed`

组合区的目标不是还原战斗节奏，而是验证静态素材叠加时仍能看清主体与信息层。

## 5. 分层规则

为避免多特效并发时互相压死，本次 Demo 采用固定分层：

### L1 格子内残留层

用于放置贴附在格子内容内部的痕迹类素材，例如：

1. 裂纹
2. 血迹
3. 压痕
4. 崩塌后的残留碎片

这层允许覆盖 Boss 部位图片，但不覆盖名称和血条。

### L2 格子主体标记层

用于放置标记型或附着型素材，例如：

1. 准星
2. 末日标记
3. 护甲破损标识
4. 盾牌碎片主体

这层贴近 Boss 主体，但要避开文本信息。

### L3 格子外爆发层

用于放置冲击、武器、弧光、外扩爆裂等大形态素材，例如：

1. 斩击轮廓
2. 镰刀残影
3. 重锤冲击片
4. 崩塌外扩轮廓

这层允许超出格子边界，负责“震撼感”。

### L4 信息保留层

保留给名字、血条、伤害信息等 UI 内容。Demo 里的静态特效不得侵入这一层。

## 6. 首版纳入的已接线特效

本次 Demo 只覆盖当前 BattlePage 已接线的视觉项：

1. `storm_combo`
2. `auto_strike`
3. `bleed`
4. `omen_harvest`
5. `final_cut`
6. `collapse_trigger`
7. `judgment_day`
8. `doom_judgment`
9. `silver_storm`
10. `crosshair-mark`

## 7. 各特效静态化方向

### 7.1 storm_combo

绿色斩击类，作为普通攻击爆发样板。放在 L3，从格子对角线掠出，不进入格子中心。

### 7.2 auto_strike

重锤打击类，拆成两部分：

1. 锤体轮廓放 L3
2. 落点压痕与碎片放 L1

### 7.3 bleed

不再使用漂浮血滴，改成贴附在格子内部的像素血迹群，放 L1。

### 7.4 omen_harvest

镰刀收割类，作为大弧线外扩特效，放 L3，占据格子外缘。

### 7.5 final_cut

终结斩击类，仍放 L3，但比普通斩击更细、更深、更集中，强调终结感。

### 7.6 collapse_trigger

作为整套视觉的结构破坏基线，拆成三段静态表达：

1. 盾牌破碎主体放 L2
2. 裂纹残留放 L1
3. 外扩爆裂轮廓放 L3

### 7.7 judgment_day

重锤裂地类，用“落点裂地 + 外扩冲击片”的双层静态构图，分别放 L1 与 L3。

### 7.8 doom_judgment

环形审判类，采用包围 Boss 部位的静态像素法环或末日环，放 L2。

### 7.9 silver_storm

不再复用镰刀图，单独设计为多段扇形残影组成的静态风暴图，主层放 L3。

### 7.10 crosshair-mark

标记类基线资源，放 L2，用于验证标记类素材在并发场景下的可读性。

## 8. 页面入口

页面以内部隐藏 Demo 页形式存在，不进入公开导航，不接真实战斗数据。入口方式沿用现有内部 Demo 思路，新增独立页面组件和本地路由判断分支。

## 9. 验收标准

完成后的 Demo 至少满足：

1. 每种已接线特效都有单独卡片。
2. 所有预览都不依赖 OSS 特效图。
3. 页面中没有动画播放。
4. 并发组合区中多个特效不会完全互相覆盖。
5. 视觉风格统一为静态像素画。

## 10. 后续落地顺序

如果 Demo 方向确认，后续正式替换可按以下顺序推进：

1. 先替换 `collapse_trigger`、`storm_combo`、`judgment_day`
2. 再替换 `auto_strike`、`final_cut`、`doom_judgment`
3. 最后收敛 `silver_storm`、`omen_harvest`、`bleed`、`crosshair-mark`
