## 1. 系统目标

- 给装备实例新增独立成长线，名称统一为“铭刻”。
- 铭刻材料独立于强化石，正式名称为“刻印石”。
- 词条写入装备实例，不污染装备原型。
- 铭刻完成后不可撤销，不提供洗词条、替换、重铸。

## 2. 材料与 Boss 掉落

新增 Boss 独立掉落配置：

- `inscriptionStoneDropRatePercent`
- `inscriptionStoneDropCountMin`
- `inscriptionStoneDropCountMax`

材料规则：

- 道具名：`刻印石`
- 每次铭刻固定消耗：`1`
- 掉落与现有装备掉落池分离，不混入 `BossLootEntry`

推荐首版：

- 新手 Boss：`8%`，掉 `1`
- 成长期 Boss：`16%`，掉 `1~2`
- 后期 Boss：`28%`，掉 `2~3`

## 3. 操作流

玩家操作：

1. 击杀 Boss，获得刻印石
2. 在背包中右键装备
3. 选择 `铭刻`
4. 阅读不可撤销确认弹窗
5. 扣除 `刻印石 x1`
6. 随机生成并写入 1 条词条

确认弹窗必须明确展示：

- `将消耗 刻印石 x1`
- `铭刻会为该装备随机新增 1 条词条`
- `铭刻后词条将与该装备永久绑定，无法撤销`
- `当前词条：X / Y`

## 4. 词条上限

铭刻词条上限与强化上限分离，按稀有度单独计算：

- 普通：`2`
- 稀有：`3`
- 史诗：`5`
- 传说：`6`
- 神话：`7`
- 至臻：`8`

兼容现状说明：

- 当前代码仍保留 `优秀` 稀有度，首版按 `普通` 的 `2` 条上限处理。

## 5. 词条池

常规词条：

- `猛攻`：攻击力 `+8 ~ +120`
- `狂烈`：暴击伤害倍率 `+0.08 ~ +0.90`
- `破势`：全伤害加成 `+2% ~ +18%`
- `屠软`：软组织增伤 `+4% ~ +30%`
- `碎甲`：重甲增伤 `+4% ~ +30%`
- `猎弱`：弱点增伤 `+4% ~ +30%`
- `奥术增幅`：魔法伤害倍率 `+4% ~ +45%`

敏感词条：

- `精准`：暴击率 `+0.15% ~ +1.20%`
- `裂甲`：额外护甲穿透 `+0.20% ~ +1.50%`
- `感电`：魔法触发率 `+0.10% ~ +0.80%`

## 6. 敏感词条约束

仅统计铭刻词条贡献的总量封顶：

- 暴击率：`6%`
- 额外护甲穿透：`8%`
- 魔法触发率：`4%`

附加规则：

- 允许重复词条
- 同名敏感词条重复时，生成权重下降
- 第二条及以后只取区间下半段
- 达到该类总量封顶后，不再生成该类词条

## 7. 数据与展示

新增词条结构：

```go
type ItemAffix struct {
    ID        string  `json:"id"`
    Name      string  `json:"name"`
    Type      string  `json:"type"`
    Value     float64 `json:"value"`
    ValueText string  `json:"valueText"`
    Tier      string  `json:"tier"`
}
```

装备实例/展示层新增：

- `Affixes []ItemAffix`
- `AffixCount int`
- `AffixLimit int`

玩家资源新增：

- `inscriptionStones`

Tooltip 规则：

- 展示 `词条：X / Y`
- 无词条时显示 `尚未铭刻`
- 有词条时逐条展示 `词条名 + 数值`

## 8. 本期边界

- 只做一种铭刻材料：`刻印石`
- 不做词条洗练
- 不做词条替换
- 不做词条拆解返还
- 不做原型级词条模板缓存系统，先直接写实例
