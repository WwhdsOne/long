## 一、结论先行

- 新增第四系天赋树：`奥术潮汐`
- 魔法体系与现有暴击率、暴击伤害、暴击系联动**完全解耦**
- 魔法系核心新增两个独立属性：
  - `magic_proc_rate`：魔法触发率
  - `magic_damage_multiplier`：魔法伤害倍率
- 魔法系总学习成本必须**高于现有三系总和**
  - 按当前成本公式粗算，现有三系总和约为 `124656`
  - 魔法系目标总成本定为 `152000` 左右
- 魔法伤害默认走“主目标 + 邻近副目标”的保守 AOE 模型
- 该方案落地时，**需要新增专属 CSS 特效与音效资源**，不能只复用现有触发表现

## 二、设计目标

### 2.1 玩法定位

`奥术潮汐` 定位为：

- 低频但稳定成长的 AOE 奢侈树
- 后期高投入专属，不应成为早期通解
- 通过“长线积累 -> 短时爆发窗口 -> 全场终结技”形成独立循环

### 2.2 明确不做的事

- 不共享现有暴击率
- 不共享现有暴击伤害
- 不让装备暴击溢出自然转化为魔法强度
- 不让普通魔法触发无限扩张目标数
- 不让终极技能在单次触发后继续连锁、补跳、二次追击

## 三、全局规则

### 3.1 新增属性

| 属性 | 含义 | 规则 |
|------|------|------|
| `magic_proc_rate` | 普通点击后额外触发魔法伤害的概率 | 默认极低，受天赋与装备影响，可设硬上限 |
| `magic_damage_multiplier` | 魔法伤害乘区 | 只作用于魔法额外伤害，不作用普通点击 |

### 3.2 魔法基础值来源

魔法伤害的基础值统一从玩家当前的**攻击力基础值**计算，不直接取“本次最终点击伤害”。

正式口径：

```text
magic_base_damage = attack_power_base_value * magic_base_ratio
magic_final_damage = magic_base_damage
                   * (1 + all_damage_amplify)
                   * magic_damage_multiplier
                   * 其他允许的魔法修正
```

其中：

- `attack_power_base_value` 指玩家当前用于普通点击结算的攻击力基础值
- 该值可以吃：
  - 攻击力成长
  - 装备基础攻击词条
  - 强化带来的攻击力成长
- 在进入最终魔法结算时，可以继续吃“全伤害加成”乘区
- `all_damage_amplify` 指当前角色面板中的全伤害加成总和
- `magic_damage_multiplier` 指魔法体系自身提供的伤害倍率
- 该值不直接吃：
  - 暴击率
  - 暴击伤害
  - 弱点暴击放大
  - 暴击系天赋联动
  - 本次点击已经结算出的最终暴击伤害

这样可以保证：

- 玩家攻击力成长会同步带动魔法成长
- 但现有暴击体系溢出不会直接把魔法也一起带爆
- 魔法体系仍保持独立可控

### 3.3 与现有体系的关系

| 系统 | 是否影响魔法 |
|------|-------------|
| 暴击率 | 否 |
| 暴击伤害 | 否 |
| 暴击系死兆 | 否 |
| 弱点暴击放大 | 否 |
| 全伤害加成 | 是 |
| 普通攻击力成长 | 是，但只通过专门折算口径进入魔法结算 |
| 装备强化 | 是，但触发率增量必须极小 |

### 3.4 强化约束

- 装备强化允许提升 `magic_proc_rate` 和 `magic_damage_multiplier`
- `magic_proc_rate` 每次强化固定只增加 `0.1%`
- `magic_damage_multiplier` 可以随强化增加，但斜率必须低于暴击伤害成长
- 由于魔法系是 AOE，任何提升都按保守斜率设计

## 四、魔法系结构与成本

### 4.1 树名与定位

- 天赋树名称：`奥术潮汐`
- 风格关键词：蓄能、裂解、潮爆、奥能、虚空、蚀刻

### 4.2 成本模型

魔法系定义为“传说树”，不沿用三系普通成本口径。

推荐 Lv1 基准成本如下：

| 层级 | 主干成本 | 填充成本 |
|------|---------|---------|
| T0 | 4200 | - |
| T1 | 5200 | 3000 |
| T2 | 7600 | 4600 |
| T3 | 10800 | 6800 |
| T4 | 26000 | - |

### 4.3 成本目标

- 整棵魔法树满配总成本目标：`152000` 左右
- 最终实现前必须按代码中的升级公式做一次精算
- 校验条件只有一条：**魔法系总成本必须严格大于三系总和**

### 5.6 T2 `回响刻印`

**作用：** 对单一目标进行长线蓄能，制造短时保底爆发窗口。

#### 机制说明

- 连续点击同一部位时累计“回响层数”
- 满层后，该部位进入 `奥术裂解` 状态
- `奥术裂解` 持续 `2 秒`
- 状态期间点击该部位时，`magic_proc_rate = 100%`
- 状态生效后清空层数
- 保底触发后进入长 CD
- 切换目标不继承层数
- 该机制只改变触发率，不改变魔法伤害倍率
- `奥术裂解` 不是暴击，不进入现有暴击体系

## 六、填充节点建议

填充节点只服务于两类目标：

- 继续微调触发率与魔法倍率
- 提供少量对单质量和副目标质量的补正

推荐节点名如下：

| 层级 | 节点名 | 作用方向 |
|------|--------|----------|
| T1 | 蓝辉 | 小幅提升魔法伤害 |
| T1 | 浮游 | 小幅提升魔法触发率 |
| T2 | 铭火 | 小幅提升主目标魔法质量 |
| T2 | 导鸣 | 小幅提升副目标承伤 |
| T3 | 深流 | 小幅提升奥术裂解期间伤害质量 |
| T3 | 渊光 | 小幅提升终极技副目标比例 |

> 本文先锁正式命名与方向。逐级具体数值在实现前统一补入开发数值表。

## 七、战斗结算口径

### 7.1 普通魔法触发

单次点击的魔法结算流程：

1. 先结算普通点击主目标伤害
2. 再按 `magic_proc_rate` 判定是否触发魔法
3. 若触发：
   - 先按攻击力基础值计算 `magic_base_damage`
   - 主目标追加一次魔法伤害
   - 邻近且存活的部位按副目标倍率承伤
4. 魔法副目标不重复命中同一格
5. 邻近无有效目标时，只结算主目标魔法伤害

补充约束：

- 普通魔法不以“本次最终点击伤害”作为母值
- 普通魔法、`奥术裂解`、`星陨潮爆` 都统一以攻击力基础值作为母值
- 魔法最终伤害明确吃“全伤害加成”乘区，保证该词条名实相符
- 终极技只是在同一母值上应用更高倍率，不额外继承暴击结算结果

建议实现表达式：

```text
主目标普通魔法伤害
= attack_power_base_value
 * magic_base_ratio
 * (1 + all_damage_amplify)
 * magic_damage_multiplier
 * 主目标额外修正

副目标普通魔法伤害
= attack_power_base_value
 * magic_base_ratio
 * (1 + all_damage_amplify)
 * magic_damage_multiplier
 * 副目标承伤比例
 * 副目标额外修正

星陨潮爆主目标伤害
= attack_power_base_value
 * magic_base_ratio
 * (1 + all_damage_amplify)
 * magic_damage_multiplier
 * 星陨潮爆主目标倍率

星陨潮爆副目标伤害
= attack_power_base_value
 * magic_base_ratio
 * (1 + all_damage_amplify)
 * magic_damage_multiplier
 * 星陨潮爆副目标比例
```

### 7.2 `奥术裂解`

- 该状态是魔法脆化窗口，不是真暴击
- 状态期间点击命中该目标，魔法一定触发
- 该状态只持续 `2 秒`
- 该状态结束后回到普通概率判定

### 7.3 终极技覆盖

`星陨潮爆` 触发时，不再使用普通邻近规则，而是直接扫描全部存活部位：

- 主目标按主倍率结算
- 其余部位按副目标比例结算
- 所有部位都应产生明确受击反馈

## 八、迁移与实现边界

### 8.1 不需要做的迁移

- 不需要对历史玩家 Redis/Mongo 数据做一次性迁移
- 原因：
  - 天赋状态本质是 `talentID -> level` 映射
  - 老玩家没有魔法系 key 时，天然等于未学习
  - 新增魔法战斗态字段时，旧数据缺失按零值处理即可

### 8.2 需要做的代码层扩展

- 把当前三系写死逻辑扩成四系：`normal / armor / crit / magic`
- 天赋接口返回第四系定义
- 战斗属性新增 `magic_proc_rate`、`magic_damage_multiplier`
- 实时协议与前端状态同步新增上述字段
- 点击结算扩成“单目标 + 多目标魔法副伤”的通用结算层
- 新增 `奥术裂解` 相关战斗态字段与冷却字段

## 九、视听表现要求

本方案落地时，**必须新增专属 CSS 特效和音效加持**，不能仅靠现有伤害跳字与通用触发音代替。

### 9.1 CSS 特效需求

至少新增以下表现：

- `奥术裂解` 状态：
  - 目标格子出现持续 2 秒的奥术脉冲边框
  - 格子表面出现轻微裂纹或流光叠层
  - 状态结束时有一次淡出或碎裂收束
- 普通魔法触发：
  - 主目标出现奥术爆点
  - 邻近副目标出现简化溅射波纹
- `星陨潮爆`：
  - 全战斗区出现一次覆盖型奥术潮汐扫场效果
  - 所有存活部位同步出现受击闪烁
  - 主目标额外叠加更强的中心爆裂效果

### 9.2 音效需求

至少新增以下音效类型：

- 普通魔法触发音
  - 短促、清脆、偏奥术，不要和现有追击/崩塌混音色
- `奥术裂解` 进入音
  - 需要有“锁定、脆化、标记完成”的听感
- `星陨潮爆` 终极音
  - 需要明显长于普通触发音，具备全场终结感
- 状态结束或爆发收束音
  - 用于把 2 秒窗口和普通状态分开

### 9.3 资源落地要求

- CSS 特效需有单独命名，不混用现有暴击、崩塌、白银风暴类样式
- 音效资源需单独命名并进入战斗触发音分类
- 前端播放逻辑应保留独立触发 key，避免和旧特效复用导致串音

## 十、数值护栏

- 魔法系总体收益必须显著慢于暴击系成型速度
- 普通魔法触发率成长要始终克制
- `回响刻印` 的 100% 触发窗口必须依赖长时间蓄能和长 CD
- `星陨潮爆` 虽打全场，但副目标比例必须长期低于主目标
- 即使装备和强化完全接入，也不允许魔法系变成“高频全屏稳定清图”

## 十一、上线前复核清单

- 魔法树总成本是否严格大于三系总和
- `回响刻印` 的命中门槛和冷却是否足够保守
- `奥术裂解` 是否完全绕开现有暴击体系
- `星陨潮爆` 的全场副目标比例是否需要继续下压
- 新 CSS 特效和音效是否已经列入实现范围，而不是留成“后补”
