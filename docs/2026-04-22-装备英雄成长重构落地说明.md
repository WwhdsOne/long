# 装备与英雄成长重构落地说明

## 本次落地规则

- 装备旧接口 `POST /api/equipment/{itemId}/synthesize` 保留，但固定返回废弃错误，不再执行业务逻辑。
- 装备成长接口改为 `POST /api/equipment/{itemId}/enhance`。
- 英雄继续使用 `POST /api/heroes/{heroId}/awaken`。
- 装备和英雄都移除旧 `reforge` / `pity` / `jackpot` / `starLevel` 语义。

## 成长公式

- 装备强化和英雄觉醒都只会命中一个基础属性。
- 命中池固定为三个基础属性：
  - 点击
  - 暴击
  - 暴击率
- 三个属性等概率命中。
- 暴击率单次成长固定为 `1/6`，后端按浮点保存，前端统一展示两位小数。
- 点击与暴击的单次成长值公式一致：
  - 当前总点击 + 当前总暴击 + 当前总暴击率
  - 将这个总和除以 `4`
  - 向上取整
  - 最小值为 `1`

示例：

- 当前属性为 点击 `2`、暴击 `0`、暴击率 `0`
- 成长基数为 `ceil((2 + 0 + 0) / 4) = 1`
- 如果本次命中点击，则结果为 点击 `3`
- 如果本次命中暴击率，则结果为 暴击率 `0.17%`

## 模板字段

- 装备模板新增 `enhanceCap`
- 英雄模板新增 `awakenCap`
- 英雄模板效果改为 `effects[]`

## 返回字段

### 装备

- `enhanceLevel`
- `enhanceCap`
- `bonusClicks`
- `bonusClicksDelta`
- `bonusCriticalChancePercent`
- `bonusCriticalChancePercentDelta`
- `bonusCriticalCount`
- `bonusCriticalCountDelta`

### 英雄

- `awakenLevel`
- `awakenCap`
- `bonusClicks`
- `bonusClicksDelta`
- `bonusCriticalChancePercent`
- `bonusCriticalChancePercentDelta`
- `bonusCriticalCount`
- `bonusCriticalCountDelta`
- `effects[]`

## Redis 清理范围

历史成长数据不迁移、不折算、不继承。运维清理时至少处理以下旧 key：

- `vote:user-equip-upgrade:*`
- `vote:user-hero-upgrade:*`

建议顺序：

1. 停写旧版本实例
2. 清理旧成长 hash
3. 发布新版本
4. 观察新字段 `enhance_level` / `clicks_delta` / `critical_chance_delta` / `critical_count_delta` 与 `effects` 是否开始写入

## 兼容说明

- 未识别的 `effect.type` 会被安全忽略，并记录 warning。
- 旧 `traitType` / `traitValue` 只保留兼容读取，不再作为主语义继续扩展。
