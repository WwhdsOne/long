## 一、核心玩法总定位

### 1.1 基础循环
玩家手动点击Boss各部位按钮造成伤害 → 部位独立血量/护甲/类型 → 全部击碎后Boss死亡 → 自动进入下一关。养成线包含装备获取与技能树点亮。

### 1.2 核心特色
| 特色         | 说明                                                         |
| ------------ | ------------------------------------------------------------ |
| 三流派克制   | 普攻均衡（绿）/ 破甲攻坚（灰）/ 暴击爆发（红），对应三类部位 |
| 统一自动战斗 | 在网页纯手动，离开页面自动接替，无切换按钮                   |
| 减法防御公式 | 护甲直接削伤害，数值透明好计算                               |
| 溢暴转化     | 暴击率溢出100%部分转为暴伤（仅暴击主系生效）                 |
| 全配置化     | Boss布局/数量/类型/属性全部后台配表驱动                      |
| 完整视觉反馈 | 三流派各自拥有独立特效体系（详见第七章）                     |

## 二、Boss部位核心规则

### 2.1 部位数量
| 场景       | 数量  |
| ---------- | ----- |
| 新手Boss   | 3个   |
| 常规Boss   | 5~7个 |
| 后期Boss   | 9个   |
| Raid级上限 | 25个  |

### 2.2 5×5坐标空间
- 固定5×5方格网格，(0,0)左上角 → (4,4)右下角
- 每格三种状态：**空位** / **常规部位**（1格，配置类型/血量/护甲） / **大型部位**（2×2+，预留）
- 默认星形布局由后台坐标JSON配置，前端按坐标动态渲染
- 25格完美对应25部位上限

### 2.3 双血量体系
- **总血量**（顶部大血条）= 所有部位血量之和
- **部位独立血量**（每个按钮自带小血条）
- 部位血量为0 → 按钮变灰、下陷、永久禁用
- 全部打爆 → Boss死亡，进入下一关

### 2.4 部位三种类型
| 类型   | 标识色 | 护甲 | 伤害系数 | 克制流派 | 默认视觉         |
| ------ | ------ | ---- | -------- | -------- | ---------------- |
| 软组织 | 绿     | 低   | 1.0      | 普攻流   | 绿色边框微光     |
| 重甲   | 灰     | 高   | 0.4      | 破甲流   | 灰色金属质感边框 |
| 弱点   | 红     | 中   | 2.5      | 暴击流   | 红色脉动光晕     |

## 三、核心伤害公式（减法防御）

### 3.1 公式
```
最终伤害 = MAX((攻击力 × 部位伤害系数 - 目标有效护甲), 1) 
         × (1 + 全伤害增幅 + Boss增伤) 
         × 暴击乘区
```

### 3.2 参数说明
| 参数                  | 说明                                   |
| --------------------- | -------------------------------------- |
| 攻击力                | 来自装备+天赋                          |
| 部位伤害系数          | 软组织1.0 / 重甲0.4 / 弱点2.5          |
| 目标有效护甲          | 原始护甲 × (1 - 破甲率)，破甲率上限80% |
| 全伤害增幅 & Boss增伤 | 来自装备+天赋，两者加法叠加            |
| 暴击乘区              | 暴击时=暴伤倍率，未暴=1.0              |
| 最低伤害              | 1（防止永远打不死）                    |

### 3.3 额外伤害类型
- **追击伤害**（普攻专属）：减护甲后直接加，无视护甲
- **真实伤害**（破甲粉碎 / 暴击出血）：直接扣血，不经公式

## 五、自动战斗系统

### 5.1 核心原则
- **不提供手动/自动切换按钮**
- 用户在网页并操作 = 纯手动
- 离开页面 = 系统自动接替
- 二者是同一进程的两种状态，非独立模式

### 5.2 状态判定
| 状态     | 条件                                     | 行为               |
| -------- | ---------------------------------------- | ------------------ |
| 手动战斗 | 页面聚焦 + 有操作                        | 纯手动点击         |
| 自动战斗 | 页面失焦<br>或页面聚焦但无操作超**30秒** | 按偏好部位自动点击 |

### 5.3 自动战斗停止条件
- 页面重新聚焦 **且** 用户产生点击/键盘操作
- 仅鼠标移动不停止

### 5.4 自动战斗执行规则
1. 实时扫描存活部位
2. 优先打偏好类型
3. 偏好类型全灭后随机打其他
4. 不锁按钮ID，换Boss自动适配，永不空点

### 5.5 状态指示器
- 页面固定位置状态灯：🟢绿色=手动 / 🟠橙色=自动战斗运行中
- 静默切换，无弹窗

## 六、装备属性体系与主系绑定

### 6.1 专精掉落
- **60%** 当前主系专精装 / **15%+15%** 另两系 / **10%** 无专精通用装

### 6.2 互斥核心词缀
| 流派       | 专精词缀（T1数值）                                           | 禁用词缀                       |
| ---------- | ------------------------------------------------------------ | ------------------------------ |
| **普攻流** | 牢固：攻击力+25%（独立乘区）<br>连击增伤：每100击部位受伤+5%<br>稳健：全伤害+15% | 暴伤倍率、溢暴转化、破甲率     |
| **暴击流** | 残暴：暴伤+60%<br>溢暴增幅：转化率→3%/1%<br>寻弱：对弱点暴击率+15% | 固定攻击力、全伤害%、破甲率    |
| **破甲流** | 贯穿：破甲率+20%<br>粉碎：对重甲附加当前护甲30%真伤<br>压制：Boss增伤+25% | 暴伤倍率、溢暴转化、固定攻击力 |

这是一个经典的掉落设计问题。给你一套完整的Boss掉落方案，包含具体数量、底层规则逻辑和扩展空间。

## Boss掉落数量设计方案

### 一、核心结论

**每次Boss击杀掉落 3~6 件装备，其中 1 件必定为该Boss最高稀有度档位。**

这个数字不是拍脑袋定的，下面拆解为什么。

---

### 二、为什么是3~6件

#### 2.1 点击游戏的时间节奏

假设一个中期Boss有7个部位，每个部位需要点击200~300次击杀，总点击约1500~2000次。手动玩家约1~2分钟击杀一个Boss，自动战斗约2~3分钟。

| 掉落件数 | 问题                                                         |
| -------- | ------------------------------------------------------------ |
| 1件      | 玩家打2分钟只看到1个掉落，反馈感太弱，尤其是白色普通装时挫败感极强 |
| 2件      | 勉强够用，但重复刷同一个Boss时经常空手而归                   |
| 3~4件    | 每次都有小收获，偶尔出好货                                   |
| 5~6件    | 收获感拉满，背包管理开始有取舍                               |
| 7件以上  | 信息过载，掉落弹窗刷屏，失去"出货"的惊喜感                   |

**3~6件是"每次都值得打，但不会溢出到麻木"的甜点区间。**

#### 2.2 稀有度分布

| 掉落件数 | 稀有度分布逻辑                       |
| -------- | ------------------------------------ |
| 3件      | 适合早期Boss，掉落集中在低稀有度     |
| 4~5件    | 中期Boss标准配置                     |
| 6件      | 后期/Raid Boss，高稀有度掉落概率更高 |

---

### 三、掉落数量与Boss阶段绑定

| Boss阶段  | 部位数量 | 掉落件数 | 最高稀有度 | 设计意图               |
| --------- | -------- | -------- | ---------- | ---------------------- |
| 新手Boss  | 3个      | **3件**  | 稀有(蓝)   | 快速熟悉掉落节奏       |
| 常规Boss  | 5~7个    | **4件**  | 史诗(紫)   | 核心推图阶段，稳定产出 |
| 高阶Boss  | 7~9个    | **5件**  | 传说(橙)   | 开始有稀有掉落期待     |
| Raid Boss | 9~15个   | **6件**  | 神话(红)   | 高投入高回报           |
| 终极Boss  | 15~25个  | **6件**  | 至臻(彩)   | 全服争夺，最高掉落规格 |

---

### 四、单次掉落的稀有度保底机制

**规则：每次Boss击杀，至少1件装备为该Boss的"最高掉落稀有度"。**

| Boss类型  | 保底稀有度 | 说明                     |
| --------- | ---------- | ------------------------ |
| 新手Boss  | 稀有(蓝)   | 保证蓝色至少1件          |
| 常规Boss  | 史诗(紫)   | 保证紫色至少1件          |
| 高阶Boss  | 传说(橙)   | 保证橙色至少1件          |
| Raid Boss | 神话(红)   | 保证红色至少1件          |
| 终极Boss  | 至臻(彩)   | 至臻不掉落则触发保底补偿 |

**为什么要保底？**
- 点击游戏的核心反馈是"打完Boss有收获"
- 如果刷10次同一个Boss全是白蓝装，玩家会觉得时间被浪费
- 保底让每次击杀都有最低限度的期待值

---

### 五、剩余掉落的稀有度分布

除去保底1件最高稀有度后，剩余掉落的稀有度按权重随机：

**常规Boss（4件掉落，保底1紫）示例：**

| 掉落序位 | 稀有度分布                              |
| -------- | --------------------------------------- |
| 第1件    | 必定史诗(紫) - 保底                     |
| 第2件    | 史诗(紫)30% / 稀有(蓝)50% / 普通(白)20% |
| 第3件    | 史诗(紫)20% / 稀有(蓝)45% / 普通(白)35% |
| 第4件    | 稀有(蓝)40% / 普通(白)60%               |

**Raid Boss（6件掉落，保底1红）示例：**

| 掉落序位 | 稀有度分布                                            |
| -------- | ----------------------------------------------------- |
| 第1件    | 必定神话(红) - 保底                                   |
| 第2件    | 神话(红)5% / 传说(橙)30% / 史诗(紫)45% / 稀有(蓝)20%  |
| 第3~4件  | 传说(橙)20% / 史诗(紫)40% / 稀有(蓝)30% / 普通(白)10% |
| 第5~6件  | 史诗(紫)30% / 稀有(蓝)40% / 普通(白)30%               |

---

### 六、流派分配规则（沿用60/30/10权重）

每次掉落的每件装备独立按60/30/10权重决定流派倾向。多件掉落时，不会出现"全是副系垃圾"的极端情况——6件独立随机，几乎必有主系。

**6件全不是主系的概率：**
- 每件非主系概率40%
- 6件全非主系 = 0.4^6 ≈ 0.4%
- 几乎不可能发生

### 七、掉落展示流程

**一次Boss击杀的掉落展示顺序：**

1. Boss死亡动画（部位全部碎裂）
2. 掉落光柱依次出现（按稀有度从低到高，最后最高稀有度压轴）
3. 光柱颜色：白→蓝→紫→橙→红→彩
4. 最后一件（最高稀有度）有专属弹出特效
5. 全部掉落汇总弹窗（3秒后自动收起，可手动点击查看详情）

玩家切走页面后Boss被自动战斗击杀，回归时：
- 弹窗显示 "Boss已被击败！"
- 列出所有掉落装备，与新Boss准备界面合并显示
- 掉落的至臻/神话装备单独高亮提示
